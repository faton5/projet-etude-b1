Optimise infrastructure Azure pour production

## Changements Infrastructure

### Backend Container App
- CPU: 0.5 → 1.0 cores (+100%)
- Memory: 1.0Gi → 2.0Gi (+100%)
- Replicas: min=0→1, max=1→3 (élimine cold starts)
- Auto-scaling: HTTP concurrency >50 req
- Workers: 1 → 2 (multi-worker Uvicorn)
- Optimisations: PYTHONOPTIMIZE=2, UVICORN_LIMIT_CONCURRENCY=100

### Frontend Container App
- CPU: 0.25 → 0.5 cores (+100%)
- Memory: 0.5Gi → 1.0Gi (+100%)
- Replicas: min=0→1, max=1→2
- Auto-scaling: HTTP concurrency >30 req
- Node.js heap: 768 MB (NODE_OPTIONS)

### Backend Dockerfile
- Multi-stage build (builder + runtime)
- Taille image: -20% (~100 MB économisés)
- Multi-worker: throughput x2
- PYTHONOPTIMIZE=2: -20% temps exec

## Scripts d'Automatisation

### scripts/update-resources.sh
Augmente CPU/RAM et configure auto-scaling
- Sauvegarde config actuelle (backups/)
- Configure min-replicas=1 (disponibilité 24/7)
- Active auto-scaling HTTP
- Coût: +26€/mois

### scripts/setup-storage.sh
Configure stockage persistant Azure Files
- Crée Storage Account + File Share (10 GB)
- Données survivent aux redéploiements
- Backup automatique géré par Azure
- Coût: +1€/mois

### scripts/setup-monitoring.sh
Active Application Insights et alertes
- Dashboard temps réel (req/sec, latence, erreurs)
- 5 alertes critiques (CPU, Memory, Restarts, Errors)
- Logs centralisés avec recherche
- Coût: +8€/mois
